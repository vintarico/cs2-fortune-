📅 Data: 26 de Outubro de 2025
🔧 Problema: Imagens das armas não apareciam na animação da roleta
✅ Status: CORRIGIDO

════════════════════════════════════════════════════════════════════════
🐛 PROBLEMA IDENTIFICADO
════════════════════════════════════════════════════════════════════════

Na animação de roleta horizontal, os cards das armas estavam aparecendo
vazios (sem as imagens PNG das skins).

CAUSA:
├─ O componente SkinImage estava recebendo className="w-24 h-24"
├─ Isso sobrescrevia as classes de tamanho padrão (size="small")
└─ As imagens não renderizavam no tamanho correto


════════════════════════════════════════════════════════════════════════
✅ CORREÇÃO APLICADA
════════════════════════════════════════════════════════════════════════

1. ARQUIVO: pages/cases/[caseId].js (Linha ~360)
   
   ANTES:
   ```javascript
   <SkinImage
     skinName={item.name}
     rarity={item.rarity}
     size="small"
     className="w-24 h-24 mb-1"
   />
   ```

   DEPOIS:
   ```javascript
   <div className="w-24 h-24 mb-1">
     <SkinImage
       skinName={item.name}
       rarity={item.rarity}
       size="medium"
     />
   </div>
   ```

   MUDANÇAS:
   ✓ Envolvido SkinImage em div com tamanho fixo
   ✓ Removido className conflitante
   ✓ Alterado size de "small" para "medium" (32x32 → melhor visibilidade)


2. ARQUIVO: components/SkinImage.js (Linha ~81)
   
   ANTES:
   ```javascript
   return (
     <div className={`relative ${sizeClasses[size]} ${className}`}>
   ```

   DEPOIS:
   ```javascript
   const appliedSize = className ? '' : sizeClasses[size];
   
   return (
     <div className={`relative ${appliedSize} ${className}`}>
   ```

   MUDANÇAS:
   ✓ Lógica para evitar conflito de classes
   ✓ Se className for fornecido, não aplica sizeClasses automático
   ✓ Permite customização total quando necessário

🐛 SE AS IMAGENS AINDA NÃO APARECEREM
════════════════════════════════════════════════════════════════════════

1. VERIFIQUE O CONSOLE DO NAVEGADOR (F12):
   └─ Procure por erros de carregamento de imagem
   └─ Mensagens: "Failed to load resource" ou "404"

2. VERIFIQUE O CAMINHO DAS IMAGENS:

3. VERIFIQUE O MAPEAMENTO:
   └─ Abra: data/weapon-images.js
   └─ Confirme que o nome da skin está mapeado corretamente
   └─ Formato: 'Nome Exato | Skin': '/skins/arquivo.png'

4. TESTE UMA IMAGEM DIRETAMENTE:
   └─ Acesse: http://localhost:3000/skins/-9a81dlW...png
   └─ Se carregar = imagem OK
   └─ Se 404 = arquivo não existe

5. LIMPE O CACHE:
   └─ Ctrl + Shift + R (hard reload)
   └─ Ou: Ferramentas > Limpar dados de navegação
